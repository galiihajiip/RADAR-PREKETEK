# Seed

Demo data for local PostGIS and Supabase production-like testing.

- `0001_demo_seed.sql` creates baseline profiles and one disaster event.
- `0002_dummy_10000_reports.sql` is generated by `npm run data:generate` and inserts 10,000 deterministic damage reports with matching image, AI prediction, and review data.

**Known issue:** this file's timestamps are hardcoded starting at `2026-07-07T01:00:00Z`, one minute per row, so the last rows land in the future relative to any "today" after 2026-07-07 (~7 days later). Since the app lists reports by `created_at DESC`, future-dated seed rows outrank real submissions made "today" and bury them off the first page. If you re-seed from scratch, either regenerate this file with a start date safely in the past, or shift it after seeding:

```sql
UPDATE damage_reports SET created_at = created_at - interval '10 days', updated_at = updated_at - interval '10 days' WHERE local_id LIKE 'local-cianjur-%';
UPDATE ai_predictions SET created_at = created_at - interval '10 days' WHERE report_id IN (SELECT id FROM damage_reports WHERE local_id LIKE 'local-cianjur-%');
UPDATE report_images SET created_at = created_at - interval '10 days' WHERE report_id IN (SELECT id FROM damage_reports WHERE local_id LIKE 'local-cianjur-%');
```
